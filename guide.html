UtilBox 운영 가이드

UtilBox는 메인 허브와 개별 실험 페이지를 분리 운영합니다. 방문자는 메인에서 전체 구조를 이해하고, 각 서브페이지에서 상세 기능을 확인할 수 있습니다.

1. 페이지 구조

  • 메인: 프로젝트 요약, 대표 페이지, 공지/광고 영역
  • 테스트 페이지: 기능 실험용 서비스 목록과 상태
  • 정책 페이지: 소개, 개인정보 처리방침, 문의, 운영 문서

2. 콘텐츠 관리 기준

  • 모든 페이지는 목적이 명확한 고유 텍스트를 포함합니다.
  • 정책/문의/소개 페이지를 상시 공개해 신뢰 정보를 제공합니다.
  • 테스트 기능은 실제 사용 흐름을 확인할 수 있도록 유지합니다.

3. 테스트 페이지 운영 원칙

실험 페이지는 오류 수정과 UX 개선이 완료되면 독립 서비스로 이전할 수 있습니다. 운영 중에는 기능 변경 내역을 기록하고, 정책 페이지와의 연결 링크를 유지합니다.

4. 광고 영역 운영 계획

광고는 콘텐츠를 가리지 않는 위치에만 배치하며, 메인/목록 페이지의 광고 슬롯을 우선 사용합니다. 서비스 품질 저하를 유발하는 과도한 배치는 지양합니다.

5. 문의 및 피드백

개선 제안 및 오류 제보는 문의 페이지를 통해 접수합니다.
